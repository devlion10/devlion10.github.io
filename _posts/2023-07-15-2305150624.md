# 오라클에서 INACTIVE 상태의 처리되지않은 락 쿼리로 인해 ACTIVE 락쿼리들을 자식처럼 연쇄적으로 양산하여 WAS(제우스)의 쓰레드풀을 차지하여 서비스에 장애.

**WAS(리눅스) SQLPLUS설치 후 쉘스크립트에서 SQLPLUS 실행**



- 쉘스크립트로 SQLPLUS 실행하여 바인드된 SQL파일 실행(tablelocked.sql:락오브젝트 및 실행쿼리 조회, vsession.sql:세션ID,시리얼, killsid.sql: 세션킬 쿼리에 vsessesion.sql에서 조회한 sid,serial 변수바인딩)

```bash
#!/bin/bash
export ORACLIENT_HOME=/data/app/sqlplus/instantclient_12_2
export PATH=$ORACLIENT_HOME:$PATH

export LD_LIBRARY_PATH=/data/app/sqlplus/instantclient_12_2
export LD_LIBRARY_PATH

cd /data/app/sqlplus/sql
echo $(date)




tn="v\$session"
echo ${tn}
locktablelist=`sqlplus -s "ID/PW@오라클IP/SID" @tablelocked.sql`

echo "locktablelist................."
echo "$locktablelist"

list=`sqlplus -s "ID/PW@오라클IP/SID" @vsession.sql`
echo "list..."
echo "$list"
if [ -n "$list" ] ; then
	echo "second exec..."
  #list 변수 바인딩(첫번째 항목만 바인딩 됨, EX:'34,453' 형태)
	res=`sqlplus -s "ID/PW@오라클IP/SID" @killsid.sql $list`
	echo ${res}
else
 	echo "empty"
fi


```




- tablelocked.sql:락오브젝트 및 실행쿼리 조회

```bash
set head off
set pages 0
set feedback off
set linesize 300

select aaa,lock_objs,curr_sql from (
        WITH s AS (
            SELECT
                ROWNUM no,
                level  lv,
                s.sid,
                s.serial#,
                s.status,
                s.sql_exec_start,
                s.event,
                s.sql_id,
                s.sql_child_number,
                s.prev_sql_id,
                s.prev_child_number,
                s.taddr,
                s.blocking_session,
                s.username,
                s.program,
                s.row_wait_obj#,
                s.row_wait_file#,
                s.row_wait_block#,
                s.row_wait_row#
            FROM
                (
                    SELECT *  FROM
                        v$session s
                    WHERE
                            1 = 1
                        AND ( ( s.type <> 'BACKGROUND'
                                AND s.status = 'ACTIVE' )
                              OR s.taddr IS NOT NULL )
                ) s
            CONNECT BY
                PRIOR s.sid = s.blocking_session
            START WITH s.blocking_session IS NULL
            ORDER SIBLINGS BY
                s.sql_exec_start NULLS FIRST
        )
        SELECT
            s.no,
            s.lv,
            ''''
            || s.sid
            || ','
            || s.serial#
            || ''''
                                                       AS aaa,
                                                       (select listagg(o.object_name,',') within group(order by o.object_name) 
        from dba_objects o, v$lock l 
        where l.id1=o.object_id and l.type = 'TM' and l.sid = s.sid) lock_objs ,
         substr(q1.sql_text, 1, 500)  curr_sql,
            s.status                                                     
        FROM
            s,
            v$transaction t,
            v$sql         q1,
            v$sql         q2,
            dba_objects   o
        WHERE
                t.addr (+) = s.taddr
            AND q1.sql_id (+) = s.sql_id
            AND q1.child_number (+) = s.sql_child_number
            AND q2.sql_id (+) = s.prev_sql_id
            AND q2.child_number (+) = s.prev_child_number
            AND o.object_id (+) = s.row_wait_obj#
            AND username = 'ROADDIG'
        ORDER BY    s.no
         )  news WHERE status = 'INACTIVE';


exit

```


- vsession.sql:세션ID,시리얼

```bash
set head off
set pages 0
set feedback off

select aaa from (
        WITH s AS (
            SELECT
                ROWNUM no,
                level  lv,
                s.sid,
                s.serial#,

                s.taddr,
                s.blocking_session,
                s.username,
                s.program,
                s.row_wait_obj#,
                s.row_wait_file#,
                s.row_wait_block#,
                s.row_wait_row#
            FROM
                (
                    SELECT *  FROM
                        v$session s
                    WHERE
                            1 = 1
                        AND ( ( s.type <> 'BACKGROUND'
                                AND s.status = 'ACTIVE' )
                              OR s.taddr IS NOT NULL )
                ) s
            CONNECT BY
                PRIOR s.sid = s.blocking_session
            START WITH s.blocking_session IS NULL
            ORDER SIBLINGS BY
                s.sql_exec_start NULLS FIRST

        SELECT
            s.no,
            s.lv,
            ''''
            || s.sid
            || ','
            || s.serial#
            || ''''
                                                       AS aaa,
            s.status

            s,
            v$transaction t,
            v$sql         q1,
            v$sql         q2,

                t.addr (+) = s.taddr
            AND q1.sql_id (+) = s.sql_id

```


- killsid.sql: 세션킬 쿼리에 vsessesion.sql에서 조회한 sid,serial 변수바인딩
